# Handoff: AI Hub — Telegram Mini App (фронтенд)

## Overview
AI Hub — агрегатор нейросетей внутри Telegram-бота: чат с текстовыми моделями (GPT-5, Claude, Gemini), генерация фото (Flux, Midjourney, DALL·E) и видео (Kling, Runway, Sora), витрина трендов, тарифы, реферальная программа, аккаунт с лимитами и настройки. Это фронтенд под **Telegram Mini App** (открывается из бота, авторизация через Telegram initData).

## About the Design Files
Файлы в этом пакете — **дизайн-референс, сделанный в HTML** (прототип, показывающий внешний вид и поведение), **а не продакшн-код для копирования как есть**. `AI Hub Redesign.dc.html` работает в специальном рантайме-песочнице (`support.js`) и не предназначен для деплоя.

Задача: **пересоздать эти экраны в реальном окружении** — рекомендую **React + Vite + TypeScript** как Telegram Mini App (либо любой стек, если уже выбран), используя нормальный роутинг, реальные `fetch`-запросы к бэкенду и Telegram WebApp SDK. Визуал (цвета, шрифты, отступы, состояния) переносить пиксель-в-пиксель по этому README и по HTML.

## Fidelity
**High-fidelity.** Финальные цвета, типографика, отступы, копирайт и интеракции заданы. Воспроизводить точно. Активное дизайн-направление — **1b «Aurora Glass»** (тёмная база + фиолетово-циановое сияние + матовое стекло). Направления 1a (Mono Brutalist) и 1c (Warm Luminous) в HTML — отклонённые альтернативы, их реализовывать НЕ нужно.

## Interactions & Behavior
- **Таббар:** переключает `tab`, сбрасывает стек и любую открытую шторку. Активная вкладка подсвечена (цвет + фон-подложка).
- **Push/Back:** Тарифы/Рефералка/Настройки кладутся в стек поверх текущей вкладки; ← снимает верхний экран. Пока стек не пуст — таббар скрыт.
- **Открытие моделей:** тап по карточке модели/действию/тренду открывает нужную шторку и подставляет модель в заголовок.
- **Выбор модели** внутри шторки: тап по сегменту меняет активную модель (подсветка градиентом) и заголовок.
- **Карусели:** горизонтальный скролл реализован вручную — **drag-to-scroll** (порог 6px, чтобы не мешать тапу) + **колесо → горизонталь**, с `stopPropagation`, т.к. в прототипе внешний холст перехватывал перетаскивание. **В обычном веб-приложении это не нужно** — используй нативный `overflow-x: auto` (+ `scroll-snap` по желанию).
- **Нажатия:** `.press` даёт `scale(.95)` на `:active`.
- **image-slot:** drag-and-drop картинки с сохранением (прототипная штука). В проде замени на обычные `<img>`/`<video>` с URL из API.

## State Management
Состояние экрана (в прототипе — один класс; в проде разложи по роутеру + стору):
- `tab`: `'home' | 'trends' | 'account'` — активная корневая вкладка.
- `stack`: массив вью поверх (`'tariffs' | 'referral' | 'settings'`); верхний элемент — текущий экран, пустой — показывается таббар.
- `overlay`: `null | 'chat' | 'image' | 'video'` — открытая шторка.
- `model`: строка выбранной модели (для заголовка шторки и запроса).
- `duration`: number (2–15) — длительность видео.
- `quality`: `'720p' | '1080p' | '4K'` — качество видео.
- **Расчёт стоимости видео** (перенести как есть): базовая цена растёт с длительностью и качеством. Формула в прототипе: `vCost = ceil(duration * qMult)`, где `qMult`: 720p→1, 1080p→1.5, 4K→3 (сверься с `renderVals()` в `AI Hub Redesign.dc.html` — там точная логика `q*Bg`, `vCost`).

## Suggested API Contract
Мини-приложение Telegram. **Авторизация:** фронт при старте берёт `window.Telegram.WebApp.initData` и шлёт его в заголовке `Authorization: tma <initData>`; бэкенд валидирует подпись (HMAC по bot token) и достаёт user_id. Никаких своих логинов.

```
GET  /api/me
     → { user: {id, username}, plan: "free|pro|premium",
         credits: 128,
         usage: { total:{used:34,limit:100},
                  fast:{used:66,limit:100}, premium:{used:12,limit:20}, images:{used:8,limit:15} } }

GET  /api/models
     → { text:[{id:"gpt-5",name:"ChatGPT",badge:"GPT-5"}, ...],
         image:[{id:"flux",name:"Flux"}, ...],
         video:[{id:"kling",name:"Kling"}, ...] }

GET  /api/trends
     → { photo:[{id,title,subtitle,model,preview_url}], video:[{id,title,subtitle,model,preview_url}] }

POST /api/chat            { model, message, conversation_id? }
     → { reply, credits_left }               // или SSE-стрим токенов

POST /api/generate/image  { model, prompt, quality:"1K|2K|4K" }
     → { job_id }
POST /api/generate/video  { model, prompt, image?, duration, quality:"720p|1080p|4K" }
     → { job_id, cost }
GET  /api/jobs/:id        → { status:"queued|running|done|error", progress?, result_url? }

GET  /api/tariffs         → { plans:[{id,name,price,period,features[],badge?}] }
POST /api/pay             { plan_id, method:"stars|card" }
     → { invoice_url }    // Telegram Stars invoice / ЮKassa redirect

GET  /api/referral        → { link, invited:7, earned:140 }

GET  /api/settings        → { lang, notifications }
PATCH /api/settings       { lang?, notifications? } → { ok:true }
```
Замечания для бэка:
- **CORS:** разрешить origin Mini App (или отдавать фронт с того же домена).
- **Платежи Stars:** обработать `pre_checkout_query` и `successful_payment` в вебхуке бота, начислить кредиты/апгрейд.
- **Генерация:** асинхронная (очередь + поллинг `/api/jobs/:id` или вебсокет); списывать кредиты при постановке в очередь, возвращать при ошибке.
- **Реферал:** `start=ref_XXXX` в deep-link боту → привязать нового юзера, начислить бонус после его первого успешного запроса.

## Files
- `AI Hub Redesign.dc.html` — прототип со всеми экранами. Актуальный, рабочий фронт — в **турне 3, опция 3a** (интерактивный телефон): вся навигация, шторки, выбор модели, карусели. Турне 2 (2a–2i) — статичные раскладки экранов по отдельности (удобно как референс каждого экрана). Турне 1 (1a/1b/1c) — сравнение визуальных направлений; реализуем **только 1b**.
- `image-slot.js` — веб-компонент слота изображения (прототипный, в проде не нужен).
- `support.js` — рантайм песочницы (НЕ переносить).

Точную логику состояния и расчёта цены смотри в блоке `<script data-dc-script>` (class `Component`) в конце `AI Hub Redesign.dc.html`.
