# Design: выверка каталога fal + качество/длительность генерации

Дата: 2026-07-15
Статус: дизайн утверждён, готов к плану реализации

## Проблема

Фронт после редизайна (`aurora-glass`) не показывает селекторы качества, которые были в
дизайн-макете (1K/2K/4K для фото, 720p/1080p/4K для видео). Они не были перенесены осознанно:
`api.generate()` таких параметров не принимает, а pydantic молча игнорирует неизвестные поля
(`app/api/routes/generate.py:31`) — селектор выглядел бы рабочим и не делал бы ничего.

При разборе выяснилось, что причина глубже фронта на четыре слоя, и по дороге вскрылись
дефекты серьёзнее исходного вопроса.

## Что выяснено (2026-07-15)

Два источника, разного веса:
- **страницы доков fal.ai** — использованы для цен;
- **машинные OpenAPI-схемы** (`https://fal.ai/api/openapi/queue/openapi.json?endpoint_id=<id>`,
  доступны без ключа и баланса) — использованы для эндпоинтов и параметров. Это контракт, а не
  описание, поэтому в спорах побеждает он: схемы **опровергли два утверждения**, полученных из
  доков (см. «Разрешения»).

### Аккаунт fal заблокирован — генерация мертва целиком

Живой вызов с рабочим ключом:
```
403 {"detail":"User is locked. Reason: Exhausted balance. Top up your balance at fal.ai/dashboard/billing."}
```
**Нулевой баланс.** Прямо сейчас любая генерация упирается в 403 независимо от каталога, цен и
качества. Блокировка срабатывает раньше валидации — поэтому проверить схемы через POST нельзя,
и все параметры ниже взяты из публичных OpenAPI-схем.

### Каталог нерабочий

`app/db/seed.py:47` честно предупреждает: `provider_model_id` и цены — плейсхолдеры,
никогда не проверявшиеся против живого API. Проверка подтвердила худшее:

Проверено запросом OpenAPI-схемы по каждому id (200 = существует, 404 = нет):

| Модель | Заявлено | Схема | Реальность |
|---|---|---|---|
| `wan_video` | `fal-ai/wan/v2.2` | **404** | **не существует**. Реально `fal-ai/wan/v2.2-a14b/text-to-video` (200) |
| `kling_video` | `fal-ai/kling-video/v2` | **404** | **не существует**. Реально `fal-ai/kling-video/v2/master/text-to-video` (200) |
| `veo_video` | `fal-ai/veo3` | 200 | существует, но **deprecated**. Преемник `fal-ai/veo3.1` (200) |
| `seedream` | `.../seedream/v3/text-to-image` | 200 | существует, но **deprecated**. Преемник `.../seedream/v4/text-to-image` (200) |
| `flux_kontext_pro` | `fal-ai/flux-pro/kontext` | 200 | существует, но схема требует `required: ["prompt","image_url"]` — это **image-to-image**. Для t2i нужен `fal-ai/flux-pro/kontext/text-to-image` (200) |
| `qwen_image`, `nano_banana`, `ovi_video` | — | 200 | эндпоинты верны |

**Важно: они падают не сразу.** Живой вызов показал, что `queue.fal.run` **принимает** запрос
к `fal-ai/wan/v2.2` (200, выдаёт `request_id`) и роняет его только в воркере:
```
POST fal-ai/wan/v2.2       → 200 IN_QUEUE → результат: {"detail":"Path /v2.2 not found"}
POST fal-ai/kling-video/v2 → 200 IN_QUEUE → результат: {"detail":"Path /v2 not found"}
```
Причина: приложение — это `fal-ai/wan`, а `/v2.2-a14b/text-to-video` — маршрут внутри него.
Заявленный `/v2.2` такого маршрута не имеет. Схема (`openapi.json?endpoint_id=…`) честно отдаёт
404, а очередь — нет.

Следствие для нашего кода: `media_generation_service` успешно поставит задачу, **зарезервирует
кредиты пользователя**, начнёт поллинг и получит эту ошибку. **Проверить, приводит ли такой ответ
к возврату кредитов** (`RequestStatus.refunded`) — иначе пользователь платит за 404. Это
отдельный денежный риск, а не косметика.

### Длительность видео: сломана у Veo, игнорируется у Wan/Ovi, работает у Kling

`FalClient.submit_video` шлёт `{"duration": <int>}` с комментарием «имя поля уточнить перед
продакшн-запуском». Реальность:

По OpenAPI-схемам + **живым вызовам** (ошибка валидации от воркера fal не стоит денег, поэтому
проверено на всех, кроме Kling — там движок принял запрос и сгенерировал, см. ниже):

| Модель | Поле длительности | Тип и значения | Что делает наш `{"duration": 5}` (int) |
|---|---|---|---|
| `kling_video` | `duration` | **string**, enum `["5","10"]`, default `"5"` | **работает**: pydantic приводит int→str. Проверено: видео сгенерировано, списано $1.40 |
| `veo3.1` | `duration` | **string**, enum `["4s","6s","8s"]`, default `"8s"` | **падает**: `literal_error: Input should be '4s', '6s' or '8s'` |
| `wan_video` | **поля нет** | длина = `num_frames` (int, default 81) + `frames_per_second` (default 16) | **молча игнорируется** — длина всегда дефолтная (81/16 ≈ 5 с) |
| `ovi_video` | **поля нет** | ни `duration`, ни `num_frames` — длина не управляется вовсе | **молча игнорируется** |

Итог: параметр «длительность» на фронте сейчас честен ровно для одной модели из четырёх.
Слайдер 2–15 сек не реализуем ни для одной: Kling умеет только 5 и 10, Veo — 4/6/8, Wan считает
кадрами, Ovi не управляется. Опасен именно средний столбец таблицы: у Wan и Ovi пользователь
двигает слайдер, платит по нему (`duration/5 × recommended_credits`) и **всегда получает 5 секунд**.

### Разрешения: единого параметра нет, но 4K существует

Здесь машинные схемы **опровергли выводы из доков**. Первичный разбор по страницам fal утверждал,
что потолок каталога — 1080p и «4K не поддерживает никто». Схемы показали обратное: доки описывали
устаревшие `veo3` / `seedream v3`, а у их преемников 4K есть.

| Модель (реальный id) | Ручка размера | Значения из схемы |
|---|---|---|
| `qwen-image` | `image_size` | `square_hd, square, portrait_4_3, portrait_16_9, landscape_4_3, landscape_16_9` или `{width,height}`; default `landscape_4_3` |
| `seedream/v3` | `image_size` | те же 6 пресетов. **4K нет** |
| `seedream/v4` | `image_size` | те же + **`auto`, `auto_2K`, `auto_4K`**; default `{2048×2048}` |
| `flux-pro/kontext` (i2i) | **только `aspect_ratio`** | 9 значений. Размером управлять нельзя |
| `flux-pro/kontext/text-to-image` | **только `aspect_ratio`** | 9 значений; default `1:1`. Размером управлять нельзя |
| `nano-banana` | **только `aspect_ratio`** | 10 значений; default `1:1`. Размером управлять нельзя |
| `ovi` | `resolution` | сырые пиксели: `512x992, 992x512, 960x512, 512x960, 720x720, 448x1120, 1120x448`; default `992x512` |
| `wan/v2.2-a14b` | `resolution` **+ `video_quality`** | `480p/580p/720p` (default `720p`) **и отдельно** `low/medium/high/maximum` (default `high`) — **две независимые оси качества** |
| `kling-video/v2/master` | **только `aspect_ratio`** | `16:9, 9:16, 1:1`. Размером управлять нельзя |
| `veo3.1` | `resolution` | `720p`, `1080p`, **`4k`**; default `720p` |

### Каталог неполон: селектор из макета описывает модель, которой у нас нет

Проверка велась по восьми моделям каталога — и это была ошибка постановки: надо было спросить,
**правильный ли сам каталог**. `fal-ai/nano-banana-pro` (Gemini 3 Pro Image, преемник обычной
Nano Banana) существует (схема 200), у неё есть и `/edit`-маршрут, и — главное —

```
resolution   string  enum=["1K","2K","4K"]  default="1K"
```

**Это буквально селектор «1K / 2K / 4K» из дизайн-макета.** Вывод спека «такой контрол
нереализуем честно ни для одной модели» был верен внутри каталога и неверен по существу:
дизайнер рисовал не фантазию, а модель, которую просто не завели.

Измерено (атрибуция чистая, без лага):

| | списано | по формуле (×2300) |
|---|---|---|
| `nano-banana-pro`, 1K (дефолт) | **$0.1500** | **345** |
| `nano-banana-pro`, 4K | **$0.3000** | **690** (×2) |
| `nano-banana` обычная | **$0.0398** | 92 (в каталоге 100 — сходится) |

Pro вчетверо дороже обычной. Цена обычной подтверждает формулу в четвёртый раз (после
`qwen_image` 46/50 и `ovi` 460/500) и заодно закрывает четвёртый знак, в котором путались
доки fal («$0.039–$0.0398» → ровно $0.0398).

Выводы:
1. **4K реализуем**: `veo3.1` (видео), `seedream/v4` (`auto_4K`) и `nano-banana-pro`
   (`resolution=4K`). Но только после ухода с депрекированных предшественников — у `veo3`
   и `seedream/v3` этих значений нет, а Pro в каталоге отсутствует вовсе.
2. **У трёх моделей ручки размера нет вообще** (`flux-pro/kontext` обоих видов, `nano-banana`,
   `kling`) — только соотношение сторон. Для них секции качества не будет, и это не недоработка.
3. **У Wan две оси**: `resolution` и `video_quality` независимы. Одна пользовательская опция
   может задавать оба поля сразу — ровно для этого `provider_params` и сделан JSONB.
4. Единый глобальный enum качества **невозможен**: контракты (`image_size` пресетами,
   `aspect_ratio`, сырые пиксели, `Np`-строки) несводимы. Решение «опции задаёт модель»
   подтверждено фактами, а не выбрано из вкуса.

### Цены: три модели из четырёх продаются в убыток

Цепочка из `calculate_text_credits`: `usd × 80 ₽ × 1.15 (provider_fee) × 2.5 (margin) / 0.10 (₽ за кредит)`.

**Все цены измерены живыми генерациями** (списание с баланса), а не взяты со страницы.
Доки fal по ценам оказались точны во всех пяти случаях.

| Модель | Цена (измерено) | Как считали | Формула даёт | В сиде | Расхождение |
|---|---|---|---|---|---|
| `qwen_image` | **$0.0200** за 1.05 МП | $0.02/МП | 46 | 50 | сходится |
| `ovi_video` | $0.20/видео (по докам, не мерили) | — | 460 | 500 | сходится |
| `wan_video` 480p | **$0.2025** за 5.06 с | $0.04/с × (81 кадр / 16 fps) | 466 | — | — |
| `wan_video` 720p | $0.405 (экстраполяция ×2 от 480p) | $0.08/с | 932 | 600 | **1.6× дешевле** |
| `kling_video` 5 с | **$1.4000** | плоско за 5 с | 3220 | 850 | **3.8× дешевле** |
| `veo_video` 4 с, звук вкл | **$1.6000** | $0.40/с | — | — | — |
| `veo_video` 4 с, звук выкл | **$0.8000** | $0.20/с | — | — | — |
| `veo_video` 8 с (дефолт), звук вкл | $3.20 (экстраполяция) | $0.40/с | 7360 | 4800 | **1.5× дешевле** |

Попутно измерение вскрыло **единицу тарификации Wan**: не «запрос», а видео-секунды,
вычисленные как `num_frames / frames_per_second` (81/16 = 5.0625 с × $0.04 = $0.2025 — сошлось
до последнего знака). Значит опции длительности Wan обязаны менять `num_frames`, и цена
провайдера поедет за ними автоматически — множитель опции должен это отражать.

Цена Kling проверена не по странице, а фактом: живая генерация 5 с списала ровно **$1.4000**
(баланс 9.9990825 → 8.5990825). Что это значит в деньгах:

| | |
|---|---|
| себестоимость | $1.40 = 112 ₽ |
| + комиссия провайдера ×1.15 | 128.8 ₽ |
| выручка: 850 кредитов по пакету START (0.149 ₽/кр) | 126.65 ₽ → **−2 ₽** |
| выручка: 850 кредитов по пакету BUSINESS (0.0856 ₽/кр) | 72.7 ₽ → **−56 ₽** |

Убыток тем больше, чем крупнее купленный пакет. Вывод «привести к формуле» подтверждён фактом.

**Veo измерен так же.** Генерация 4 с / 720p / звук по умолчанию списала ровно **$1.60**
(7.1990825 → 5.5990825), то есть **$0.40 за секунду**. Это разрешает противоречие в доках:
верен набор «$0.20 без звука / $0.40 со звуком», а «$0.50/$0.75 standard» — неверен.
Открытый вопрос №1 закрыт фактом. Нюансы:
- **`generate_audio: true` по умолчанию, и это ровно удваивает цену — ПРОВЕРЕНО.**
  Две генерации, одинаковые во всём кроме звука:
  | конфигурация | списано |
  |---|---|
  | 4 с / 720p / `generate_audio` не передан (→ true) | **$1.60** ($0.40/с) |
  | 4 с / 720p / `generate_audio: false` | **$0.80** ($0.20/с) |

  Разделение подтверждено: пакет «Veo без звука + qwen_image» списал $0.82, а отдельный замер
  картинки дал ровно $0.02 → на Veo приходится $0.80.

  Методическая оговорка на будущее: списания у fal **запаздывают неравномерно** — у Kling
  мгновенно, у Veo и qwen_image десятки секунд. Замеры подряд слипаются и дают ложный $0.00.
  Мерить надо, дожидаясь стабилизации баланса, а не сразу после ответа.

  Наш `submit_video` шлёт только `{prompt, duration}`, то есть звук включается молча. Ни во
  фронте, ни в API его нет; результат пользователь смотрит в `<video>` — вероятно, без звука
  вовсе. **Мы платим вдвое за озвучку, которую никто не заказывал.**
  **Это ещё одна опция для `model_option`** (`kind=audio`, либо частью quality-опции с
  `credits_multiplier: 0.5` на «без звука»). Экономия — 50% себестоимости самой дорогой
  модели каталога, без потерь для пользователя.
- **`recommended_credits` «за 5 секунд» для Veo бессмысленны**: модель принимает только 4/6/8 с.
  При её дефолте 8 с реальная себестоимость — $3.20, формула даёт 7360 кредитов против 4800
  в сиде. Убытка нет (4800 кр по START ≈ 715 ₽ против 294 ₽ себестоимости), но маржа вдвое
  ниже задуманной.
- В отличие от Kling, списание у Veo **запаздывает** на десятки секунд. Для тестов это значит:
  мерить баланс не сразу, иначе получишь $0.00 и ложный вывод «бесплатно».

`qwen_image` и `ovi` сходятся с формулой почти точно — значит формула и есть задуманный способ
расчёта, а `wan`/`kling`/`veo` посчитаны по неверным (устаревшим?) ценам провайдера.

**Kling убыточен**: 850 кредитов × 0.149 ₽/кредит (пакет START) = 127 ₽ выручки против
себестоимости $1.40 ≈ 112 ₽, с комиссией провайдера 128.8 ₽ — минус на каждой генерации.

## Принятые решения

1. **Выверка каталога и качество — один спек**, каталог первым этапом.
2. **Опции задаёт сама модель, в БД.** Никаких глобальных enum'ов качества: фронт рисует то,
   что модель реально умеет. У `nano_banana` секции качества не будет вовсе.
3. **Цена опции — множитель** (подход A), а не расчёт из себестоимости (C). Причина: достоверных
   USD нет (`fixed_cost_usd = 0` у всех, по Veo противоречивые данные), а множитель переиспользует
   существующие формулы без правок и выражает нелинейные цены провайдера, которые формулой не
   выражаются. Дорога к C открыта: когда USD появятся, множитель заменяется расчётом без ломки схемы.
4. **Цены приводятся к формуле** (wan/kling/veo пересчитываются). Следствие — см. «Последствия».

## Рассмотрено и отклонено: OpenRouter вместо fal для медиа

Проверено живыми вызовами 2026-07-15 (в проекте уже есть рабочая интеграция —
`app/services/ai/openrouter_service.py`, `ModelProvider.openrouter`).

**Видео — невозможно.** Из 342 моделей каталога OpenRouter (`GET /api/v1/models`) моделей с
`architecture.output_modalities` содержащим `video` — **ноль**. Kling/Veo/Wan/Ovi заменить нечем.

**Фото — работает, но убивает задачу.** 9 моделей с выходом `image`
(`google/gemini-3.1-flash-image`, `google/gemini-3-pro-image`, `openai/gpt-5-image`, …).
Контрольная генерация прошла успешно: PNG 1408×768, `usage.cost = $0.067205`,
`image_tokens = 1120` — при том что fal в это же время отвечал 403 (нулевой баланс).
Но у **всех** image-моделей OpenRouter в `supported_parameters` нет ни `image_size`, ни
`resolution`, ни `aspect_ratio` — только `max_tokens`, `temperature`, `seed`, `response_format`,
`top_p`, `reasoning`. Разрешение выбирает модель (1408×768 в тесте никто не заказывал).
**Селектор качества стал бы невозможен в принципе**, тогда как у fal `qwen_image` даёт
`image_size`, а `seedream/v4` — `auto_2K`/`auto_4K`.

Решение: **остаёмся на fal**, баланс пополняется (этап 0). Исходная задача — вернуть селекторы
качества — решается только там.

Два наблюдения на будущее, чтобы не потерялись:
1. `nano_banana` у fal — это Gemini-шная image-модель Google, которую OpenRouter отдаёт напрямую
   по той же цене (~$0.039). fal в этой позиции — посредник без добавленной стоимости.
2. **OpenRouter возвращает фактическую себестоимость** в ответе (`usage.cost`,
   `cost_details.upstream_inference_cost`), а fal — нет. Для OpenRouter-моделей это делает
   возможным отклонённый здесь подход C (считать кредиты от реального расхода с маржой) — та
   проблема, из-за которой Kling продаётся в убыток, для них не возникает by design.
   Если медиа когда-нибудь поедет на OpenRouter, цену считать оттуда.

## Архитектура

### Данные

Новая таблица `model_option`:

| Колонка | Тип | Смысл |
|---|---|---|
| `id` | PK | |
| `model_id` | FK → `ai_models.id`, ON DELETE CASCADE | владелец опции |
| `kind` | enum (`quality`, `duration`) | вид параметра |
| `code` | String(32) | стабильный код для API (`"720p"`, `"10s"`) |
| `label` | String(64) | что видит юзер (`«720p»`, `«10 сек»`) |
| `provider_params` | JSONB | что уходит провайдеру (см. ниже) |
| `credits_multiplier` | Numeric(6,3), default 1.0 | множитель цены относительно дефолта |
| `is_default` | Boolean | подставляется, если клиент не выбрал |
| `sort_order` | Integer | порядок сегментов |
| `is_active` | Boolean | скрыть без удаления |

Ограничения:
- `UNIQUE (model_id, kind, code)`
- **Ровно один `is_default = true` на пару (model_id, kind)** — частичным уникальным индексом
  (`CREATE UNIQUE INDEX ... ON model_option (model_id, kind) WHERE is_default`), а не соглашением.
- Индекс `(model_id, kind, sort_order)` — по нему читает `/api/models`.

`provider_params` — источник правды по контракту провайдера. Именно поэтому JSONB: контракты
несводимы между собой.

Значения ниже сверены с OpenAPI-схемами:

| Модель | kind | code | provider_params |
|---|---|---|---|
| `wan_video` | quality | `480p` | `{"resolution": "480p", "video_quality": "high"}` — **одна опция задаёт две оси** |
| `wan_video` | quality | `720p` | `{"resolution": "720p", "video_quality": "maximum"}` |
| `wan_video` | duration | `5s` | `{"num_frames": 81, "frames_per_second": 16}` — совпадает с дефолтом модели (81/16 ≈ 5.06 с) |
| `wan_video` | duration | `10s` | `{"num_frames": 161, "frames_per_second": 16}` (161 — верхняя граница схемы) |
| `kling_video` | duration | `10s` | `{"duration": "10"}` — **строка**, не int |
| `veo_video` | duration | `8s` | `{"duration": "8s"}` — **строка с суффиксом** |
| `veo_video` | quality | `4k` | `{"resolution": "4k"}` — только у `veo3.1` |
| `ovi_video` | quality | `992x512` | `{"resolution": "992x512"}` — сырые пиксели |
| `qwen_image` | quality | `2k` | `{"image_size": {"width": 2048, "height": 2048}}` |
| `seedream` | quality | `4k` | `{"image_size": "auto_4K"}` — только у `v4` |
| `nano_banana`, `flux_kontext_pro`, `kling_video` | quality | — | **опций нет**: у провайдера нет ручки размера |

Строка Wan показывает, зачем нужен JSONB, а не колонка: одна пользовательская опция «720p»
выставляет два независимых поля провайдера. Строка `kling`/`veo` показывает, зачем нужен именно
JSON, а не строка: типы значений (`"10"` строкой, `161` числом) обязаны сохраниться.

### Цена

Опции — **множительный слой поверх** существующих формул. `recommended_credits` становится ценой
**дефолтной комбинации** опций.

```
base     = существующая формула (calculate_image_credits / calculate_video_credits)
credits  = ceil(base × Π credits_multiplier выбранных опций)
credits  = max(credits, model.min_credits, VIDEO_MIN_CREDITS для видео)
```

**Порядок обязателен**: множитель применяется ДО минимумов, иначе дешёвая опция пробьёт пол и
станет бесплатной, а дорогая — недоплаченной.

Для видео формула `duration_seconds / VIDEO_BASE_SECONDS` **удаляется**: её работу берёт множитель
опции длительности. Это и позволяет выразить нелинейность Kling (`"10"` → ×2.0, потому что $2.80
против $1.40 — а не ×2 по числу секунд).

Для `qwen_image` (`cost_unit = megapixel`) `megapixels` остаётся 1.0 в вызове, а разрешение
выражается множителем опции (2K → ×4.0). `cost_unit` сохраняется — он всё ещё описывает базис
себестоимости в `calculate_*_api_cost_usd`.

### Провайдер

`FalClient.submit_image` / `submit_video` перестают конструировать `body` из угаданных полей:

```
body = {"prompt": prompt}
     + {"image_url": ...} если есть
     + merge(provider_params всех выбранных опций)
```

Адаптер больше не знает про `duration`, `resolution` и `num_frames` — он мержит то, что лежит в БД.
PLACEHOLDER-комментарий про имя поля длительности удаляется вместе с угадыванием.

### API и фронт

- `GenerateRequest` += `quality_code: str | None`, `duration_code: str | None`. **Коды опций,
  не сырые значения** — иначе клиент сможет прислать произвольный `num_frames`.
- `ModelOut` += `options: list[ModelOptionOut]` (`kind`, `code`, `label`, `credits_multiplier`,
  `is_default`, `sort_order`), только `is_active`.
- Фронт (`/generate-image`, `/generate-video`) рисует `SegmentedControl` **из пришедших опций**.
  Нет опций данного `kind` → секции нет (у `nano_banana` качества не будет).
  Слайдер длительности 2–15 удаляется, заменяется сегментами.
- Цена в CTA пересчитывается локально умножением. Точную сумму по-прежнему даёт 409-гейт.

### Ошибки

- Неизвестный / неактивный / чужой `code` → **400**, не тихий дефолт. Тихий откат вернул бы нас
  ровно к тому, от чего уходим.
- Код не передан, а у модели есть опции этого `kind` → берётся `is_default`.
- `provider_params` валидируется при записи в админке, не в горячем пути генерации.
- 409-гейт (`ConfirmationRequiredError`) не меняется: сумма считается до резервирования, значит
  уже с учётом множителей.

## Этапы

**Этап 0 — пополнить баланс fal.** Аккаунт заблокирован по нулевому балансу (см. «Аккаунт fal
заблокирован» выше). Без этого ничего из перечисленного ниже нельзя проверить вживую, а продавать
пользователям — нельзя вовсе. Это блокер уровня всего спека, а не этап реализации.

**Этап 1 — каталог** (без него остальное бессмысленно):
- миграция + сид: правка `provider_model_id` на проверенные схемой id
  (`fal-ai/wan/v2.2-a14b/text-to-video`, `fal-ai/kling-video/v2/master/text-to-video`),
  разведение `flux_kontext_pro` на t2i (`fal-ai/flux-pro/kontext/text-to-image`) и
  i2i (`fal-ai/flux-pro/kontext`, где `image_url` обязателен по схеме);
- миграция с депрекированных на `seedream/v4` и `veo3.1` — **без неё 2K/4K недостижимы**,
  то есть исходная задача не решается;
- пересчёт `recommended_credits` по формуле, все основания измерены:
  **`kling` → 3220** ($1.40 за 5 с), **`veo` → 7360** (8 с со звуком, $0.40/с) либо **3680**
  без звука, **`wan` 720p → 932** ($0.08/с × 5.0625 с; 480p измерен как $0.0405/с, 720p —
  удвоение по докам, которые в ценах не ошиблись ни разу).
  Выключать `veo_video` больше не нужно — вопрос по его цене закрыт фактом;
- `min_credits` пересматривается вместе с `recommended_credits`: сейчас у kling `min_credits = 850`,
  и после подъёма базы до 3220 старый минимум становится бессмысленным (пол ниже честной цены
  дефолтной комбинации). Правило: `min_credits` ≤ цена самой дешёвой комбинации опций.

**Этап 2 — `model_option`**: миграция, модель, сид опций по таблице выше.

**Этап 3 — цена**: множители в `pricing.py`, удаление `duration/5` для видео.

**Этап 4 — адаптер**: merge `provider_params`, удаление угаданных полей.

**Этап 5 — API**: `quality_code`/`duration_code`, `options` в `ModelOut`, 400 на неизвестный код.

**Этап 6 — фронт**: сегменты из опций, удаление слайдера, пересчёт цены.

**Этап 7 — админка**: CRUD опций (паттерн `AdminModels`).

## Тесты

- `pricing`: множители; **порядок применения относительно минимумов**; нелинейность Kling
  (5с → 3220, 10с → 6440); дефолтная комбинация == `recommended_credits`.
- Адаптер: в body уходит ровно `provider_params`, **с сохранением типов** — `"5"` строкой,
  `"8s"` строкой. Именно на типах всё и сломалось.
- API: 400 на неизвестный/неактивный код; подстановка `is_default` при отсутствии.
- БД: частичный уникальный индекс не даёт завести два дефолта.
- e2e: модель без опций → селектор не рендерится; смена опции меняет цену в CTA.

## Найденные денежные баги (вне исходной задачи, но вскрылись по дороге)

1. **Переплата за длительность у Wan и Ovi.** `calculate_video_credits` для `cost_unit=second`
   берёт `ceil(duration_seconds / 5 × recommended_credits)`. Пользователь ставит слайдер на 15 с
   и платит 1800 кредитов вместо 600 — но у Wan поля длительности нет, `duration_seconds`
   уходит в никуда, и приходит **всегда 5-секундное видео**. Тройная цена за то, что мы даже
   не запрашиваем у провайдера. Лечится этим спеком (опции длительности вместо слайдера),
   но стоит знать, что это уже происходит.
2. **Kling продаётся ниже себестоимости** — см. таблицу выше, подтверждено списанием.
3. **Резерв кредитов под 404.** Два сломанных эндпоинта падают после постановки в очередь,
   когда кредиты уже зарезервированы. Требует проверки пути возврата.

## Последствия (не замалчивать)

**Пакеты кредитов ломаются.** После приведения цен к формуле START (1000 кредитов за 149 ₽) не
покрывает одну генерацию Kling (3220). Видео станет доступно только с пакета PLUS (12000).
Требует отдельного продуктового решения — в этот спек не входит, но всплывёт сразу.

## Открытые вопросы

1. ~~Цена Veo недостоверна~~ — **ЗАКРЫТО ФАКТОМ**: генерация 4 с / 720p / звук=default списала
   ровно $1.60 → **$0.40/с**. Верен доковый набор «$0.20 без звука / $0.40 со звуком»;
   «$0.50/$0.75» — неверен. `veo_video` можно включать и пересчитывать (7360 кредитов при
   дефолтных 8 с со звуком; вдвое меньше — при звуке выключенном).
2. ~~Длительность Ovi~~ — **ЗАКРЫТО**: в схеме `OviInput` нет ни `duration`, ни `num_frames`.
   Длина не управляется. У `ovi_video` опций `kind=duration` не будет.
3. ~~Живы ли deprecated~~ — **ЗАКРЫТО**: `seedream/v3` и `veo3` оба принимают запросы и
   валидируют их, то есть работают. Но мигрировать всё равно надо: **4K и 2K доступны только
   у преемников** (`veo3.1`, `seedream/v4`) — живой вызов подтвердил, что enum `image_size`
   у v3 не содержит `auto_4K`, а `veo3.1` принимает `resolution: "4k"`. Без миграции исходная
   задача («вернуть селекторы качества») нереализуема в полном объёме.
4. ~~Влияет ли разрешение на цену у `veo3.1`~~ — **ЗАКРЫТО ЗАМЕРОМ**. Три генерации 4 с без звука:

   | разрешение | списано | множитель |
   |---|---|---|
   | 720p (дефолт) | $0.80 | 1.0 |
   | **1080p** | **$0.80** | **1.0 — бесплатное улучшение** |
   | **4k** | **$1.60** | **2.0** |

   Неочевидный результат: 1080p стоит столько же, сколько 720p — то есть можно отдавать его
   даром. Доки о разнице 720p/1080p не говорили вовсе, а «три градации = три цены» оказалось
   неверным. Дорожает только 4K, ровно вдвое.

   **Композиция множителей подтверждена данными**: 4 с без звука = $0.80, 8 с со звуком = $3.20
   = 0.80 × 2 (длительность) × 2 (звук). Оси перемножаются — ровно так, как считает резолв опций.
   Оговорка: композиция *разрешения* со звуком не проверена (4k+звук стоил бы $3.20 при балансе
   $2.16). Умножение здесь — обоснованное допущение по аналогии, не факт.

   **`seedream/v4` тоже измерен**: три генерации (`square_hd`, `auto_2K`, `auto_4K`) списали
   ровно $0.09 суммарно, то есть **$0.03 за картинку независимо от разрешения**. Сходится с
   `cost_unit=image` (плоская цена). Множители всех трёх = 1.0 — 2K и 4K бесплатны.
   (Атрибуция по отдельным замерам смазана запаздывающим списанием, но сумма однозначна:
   будь 4K дороже, итог превысил бы $0.09.)

   **Итог по разрешениям: платно оно только там, где провайдер считает по объёму** —
   Wan (per-second, тариф зависит от resolution) и qwen_image (per-megapixel). Там же, где
   тариф плоский за единицу (Seedream — за картинку, Veo — за секунду), разрешение
   бесплатно вплоть до 4K у Seedream и до 1080p у Veo. Veo 4k — единственное исключение:
   плоский посекундный тариф, но 4k его удваивает.
5. ~~Цены только по докам~~ — **ЗАКРЫТО**: измерены `kling` ($1.40), `veo` со звуком ($0.40/с)
   и без ($0.20/с), `wan` 480p ($0.0405/с фактически = $0.04/с × 5.0625 с), `qwen_image`
   ($0.02/МП). **Доки fal по ценам точны во всех пяти случаях.** Не измерены: `ovi` ($0.20
   плоско), `seedream`, `flux_kontext`, `nano_banana`, `wan` 720p.

   Наблюдение, важное для доверия к источникам: доки fal **безупречны в ценах и ошибочны в
   возможностях** (4K, типы `duration`, наличие полей). Ошибались они там, где описывали
   устаревшие версии (`veo3` вместо `veo3.1`, `seedream v3` вместо `v4`) — врал не документ,
   а его давность. Прайс обновляют, описание возможностей — нет.
